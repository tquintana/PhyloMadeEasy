Maximum Likelihood Method


The Maximum Likelihood (ML) method is similar to the MP method in that it creates a phylogenetic tree based on the sites within the sequences of interest. However, the ML method focuses on the likelihood that an unrooted tree produced from the nucleotides or amino acids found at each site will occur in the final most-likely phylogenetic tree.

The ML algorithm is as follows:

  1. For each site within the multiple sequence alignment, produce an unrooted tree containing the nucleotides/amino acids found at that site.




  2. For each of the unrooted trees, calculate the likelihood $L_{j}$ at each site $j$ from a chosen model of substitution, such as one which gives the probability for nucleotide substitution based on the length of the connecting branch. Only the points of the tree, that is, the nodes which are at the head of each clade, are used to calculate the likelihood. Be sure to use a model that is specific to the multiple sequence alignment and the resulting unrooted trees.




  3. Calculate the total likelihood of the tree using the following equation, where $L$ is the total likelihood and $n$ is the number of sites within the multiple sequence alignment:

    $L = \sum_{j = 1}^{n} L_{j}$


  4. The tree with the greatest likelihood is considered the "best" tree, and is chosen as the output.

The ML method is one of the tree-building methods which has a base within statistical analysis, as maximum-likelihood analysis is used in estimating a statistically relevant model for a dataset. Also, the ML method is the only method presented which uses all the data provided by the multiple sequence alignment. Ultimately, when there are no informative sites within the multiple sequence alignment to use the MP method, the ML method is then used to create a more accurate tree than distance-based methods, although at a much higher time cost.

The time complexity for the ML method is unknown, as there are many possible combinations of data (different models of substitution, whether the most-likely tree is unknown, etc.) with which to calculate the most-likely tree.
